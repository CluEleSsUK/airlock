# Host dependencies installed for airlock

What was added to this machine while building airlock, so you can reproduce or roll
it back. Almost everything airlock needs was already present.

## Installed by me

- **smolvm 1.3.3** — the microVM engine airlock orchestrates. Installed from the
  pinned GitHub release, checksum-verified, not `curl | bash`:
  - Archive extracted to `~/.local/opt/smolvm-1.3.3-linux-x86_64/`
  - Symlinked onto `PATH`: `~/.local/bin/smolvm` → that directory's `smolvm` wrapper
  - The wrapper bundles libkrun/libkrunfw and the agent rootfs; no other system
    packages were needed.
  - Rollback: `rm ~/.local/bin/smolvm ~/.local/opt/smolvm-1.3.3-linux-x86_64 -rf`

That is the only host dependency I installed.

## Already present (used, not installed)

`docker` + `podman` (image build), `qemu`/KVM (`/dev/kvm`, user in `kvm` group),
`passt` + `slirp4netns` (smolvm networking), `go` 1.26, `node` 26, `rust`/`cargo`
1.95, `gh` (authenticated), `ssh`/`ssh-keygen`, `git`.

## Transient artifacts (safe to delete)

- `~/.local/share/airlock/<profile>/` — per-profile SSH keypair, build context,
  and the ~2 GB `image.tar`. Regenerated by `airlock build`.
- `~/.local/state/airlock/<profile>/fleet.json` — the fleet index.
- Docker images tagged `airlock/<profile>:latest` and the base `ubuntu:24.04`.
- smolvm machine state under smolvm's own data dir (removed by `airlock rm`).

## Rust crate dependencies (project, pinned in Cargo.lock)

`clap`, `serde`/`serde_json`, `toml`, `thiserror`, `anyhow`, `tracing` +
`tracing-subscriber`, `directories`, `dotenvy`, `which`; `tempfile` (dev). No
network/crypto crates — key generation shells out to the system `ssh-keygen`.
